# Sync

`npm run sync` composes the shared rules in `rules/` into per-agent configuration
files.

## Inputs

- `rules/manifest.json` — ordered list of rule files.
- `rules/*.md` — the rule content.

## Outputs

| Agent | File(s) |
|---|---|
| Cross-agent | `AGENTS.md` |
| Claude Code | `CLAUDE.md`, `.claude/settings.json` |
| Cursor | `.cursor/rules/sdd_template.mdc` |
| Codex | `.codex/instructions.md` |
| Gemini CLI | `GEMINI.md`, `.gemini/settings.json` |
| OpenCode | `.opencode/instructions.md` |
| Grok Build | `.grok/instructions.md` |
| GitHub Copilot | `.github/copilot-instructions.md` |
| Docs | `docs/agents.md` |

## Headers and managed blocks

Every generated markdown file begins with:

```text
<!-- AUTO-GENERATED by sdd_template. Do not edit directly.
Source: rules/, skills/, hooks/, mcp-configs/
Run: npm run sync
-->
```

Managed content is wrapped in markers:

```md
<!-- BEGIN MANAGED AGENT RULES -->
...generated content...
<!-- END MANAGED AGENT RULES -->
```

For markdown files, content outside the managed markers is preserved on re-run,
so you can add project-specific notes around the managed block.

## Modes

- `npm run sync -- --dry-run` — preview planned files, write nothing.
- `npm run sync -- --check` — validate planned files (e.g. header presence).
- `npm run sync` — write the files.
