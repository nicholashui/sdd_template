Welcome to Everything Claude Code, the complete field guide. This is a book-length, hands-on guide to a project called ECC, short for Everything Claude Code. Think of it as an agent-harness operating system for AI-assisted software development.

Over the course of this guide, you'll learn what ECC is, where it came from, how it's built, and most importantly, how to actually use it day to day. It's written for someone who has never touched the project before and wants to go from zero to confident operator.

Before we go any further, here's an important heads-up about the name. If you came here searching for ECC expecting elliptic curve cryptography, this is a completely different project. The short repository name redirects to Everything Claude Code. So throughout this entire book, when we say ECC, we mean Everything Claude Code. There's no cryptography here at all. It's a system of agents, skills, hooks, and rules that supercharge AI coding assistants.

So who is this book for? It's for new users who installed ECC, or are thinking about it, and feel overwhelmed by sixty-four agents, two hundred and sixty-two skills, and eighty-four commands. It's for existing Claude Code, Cursor, or Codex users who want a clear mental model instead of a pile of config files. It's for team leads evaluating whether to adopt ECC and how to do it safely. And it's for anyone simply curious about how a serious agent harness is engineered.

No prior ECC knowledge is assumed. It helps if you're comfortable with a terminal and have used at least one AI coding tool, whether that's Claude Code, Cursor, Codex, or something similar, but that's all you need.

Now, how should you read this book? The chapters are ordered so each one builds on the last, but they're also self-contained enough to jump around. If you're in a hurry, here are a few shortcuts. If you just want it working, read the preface, then the installation chapter, then the everyday workflows chapter. If you want to understand it first, read the background and architecture chapters, then dive into the components. And if you're worried about safety, read the security chapter before you install anything that runs hooks.

Here's a quick map of where we're headed. Part one is orientation, covering what ECC is and why it exists. Part two gets it running, with installation and core concepts. Part three is the heart of the book, the six building blocks: agents, skills, commands, hooks, rules and memory, and MCP with context management. Part four is about operating ECC, including everyday workflows, cross-harness use, continuous learning, and token optimization. Part five covers safety and advanced topics, including security, ECC two-point-oh and its command-line interface, and the dashboard and tooling. And part six is reference material, with troubleshooting, a frequently asked questions section, and a glossary.

One last note on accuracy and sources. Everything in this book was written after directly reading the ECC source tree at version two-point-oh-point-oh. ECC ships updates weekly, so exact counts of agents, skills, and commands, along with command names, drift over time. Always cross-check against the live repository and the guides inside it. The project is released under the MIT license. The content in this book was summarized and rephrased from the ECC repository and its documentation for teaching purposes.
