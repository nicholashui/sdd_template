第零章。前言和快速上手。

我们就从两分钟版本开始。你会用 AI 编程助手。可能是 Claude Code，可能是 Cursor，可能是 Codex 或者 OpenCode，总之就是那一类东西。它开箱即用，是一个聪明但健忘的通才。它每一次会话都要重新探索你的代码库。它会忘记你昨天教过它的惯例。它会高高兴兴地 commit 一句多余的调试日志，而且时不时还会做一些有风险的事，因为根本没有任何东西阻止过它。

Everything Claude Code，也就是 ECC，是一个单一、可重用的层，你把它放在那个助手前面，就可以一次性修正以上所有问题。它给了助手以下这些东西。

第一，可以委派工作的专家。一共有 64 个 agents，包括 planner、code reviewer、security reviewer、build-error resolver，还有很多其他的。

第二，可重复的工作流程。一共有 262 个 skills，涵盖像测试驱动开发、安全审查和 API 设计这样的东西。

第三，自动护栏。这些是 hooks，帮你格式化代码、扫描机密信息，以及保存你的上下文。

第四，常驻标准。这些是 rules，涵盖不可变性、八成的测试覆盖率、惯例式 commit，以及不要硬编码机密信息。

第五，可以跨会话留存的记忆，再加上一个学习循环，把重复出现的模式转化成全新的 skills。

第六，安全，包括沙箱指引和一个内置的扫描器，叫做 AgentShield。

而且它可以在很多款工具上做到这些。你只需编写一次，ECC 就会把它翻译成每个框架的原生配置格式。

如果你看完整本书只记得一句话，那就请记住这句。ECC 会把一个聪明但通用的 AI 助手，变成一位有纪律的资深工程师，配备一支专家团队、一份记忆，以及一名保安。

现在我们讲清楚 ECC 不是什么。它不是椭圆曲线密码学。简短的 repo 名只是重定向到 Everything Claude Code。它不是一个模型，也不是一个 API。它是跑在你已经在用的 AI 工具之上的。它也不是锁死在 Claude 上。虽然名字是这样，但它支持 Cursor、Codex、OpenCode、Gemini、Zed、GitHub Copilot 等等。

我们试一下在五分钟之内用它，用 Claude Code 做例子。完整安装、其他选择和其他框架，后面再讲，这里只是最快的顺利路径。

第一步。在 Claude Code 里面，加入 marketplace 并安装插件。你加入指向 ECC repo 的 marketplace，然后安装 ecc 这个插件。

第二步。加入你关心的 rules。插件无法附带 rules，所以你要手动复制。你 clone ECC 的 repo，在你的 Claude 配置下面建一个 rules 文件夹，然后把 common 的 rules，加上你技术栈的 rules，例如 TypeScript，复制进去。

第三步。用它。你执行 plan 这个 command，加上一句描述，例如：用 OAuth 加入用户验证。

这就是整个循环。你描述你的意图，ECC 帮你规划，委派给合适的专家，强制执行测试和标准，并且记住发生过的事。

不确定要装什么？你可以在任何项目里问一问内置的顾问。有一个 consult command，会建议你应该配置什么，例如问它关于安全审查的事，并且指定目标是 Claude。

现在，这里是安装 ECC 的黄金法则。安装有两条路线：插件，和手动安装器。挑其中一条就够了。最常见的那种搞坏了的配置，就是把两者叠在一起，先装插件，然后又执行完整安装器，这样就会让 skills 和 hooks 重复。我们后面会再三强调这一点，但你现在就要记在心里。

这里是本书其余部分的流程。第一部分是入门定向，讲是什么和为什么。第二部分让它运作起来，涵盖安装和概念。第三部分讲各个构建模块，从 agents 一直到 MCP。第四部分讲通过日常工作流程去操作 ECC。第五部分讲安全和进阶主题，包括安全和 ECC 2.0。而第六部分是参考资料，有疑难排解和词汇表。
