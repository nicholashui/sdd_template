Chapter sixteen. ECC two-point-oh and the operator command-line interface.

So far, we've treated ECC as a catalog you install into one harness. ECC two-point-oh introduces a layer above that: a way to operate many agent sessions from one surface. This chapter covers the Rust control-plane prototype and the operator command-line interface you can use today.

Let's start with the motivation. Once you're running several agents, across different worktrees, different harnesses, some looping autonomously, you need answers to operator questions. Which sessions are running? Which stalled? Which need review? What's the risk? And where did I leave off? That's the job of a control plane. ECC two-point-oh is the layer above individual harness installs: it manages many agent sessions, keeps state, output, and risk visible, and adds orchestration and review controls. It's Claude Code first, without blocking future harness interoperability.

The current ECC two-point-oh scaffold is a Rust-based control plane, and the repo is honest about its status, so you should be too. It is real code, alpha quality, valid to build and test locally. But it is not the finished ECC two-point-oh product. The repo even has a rule: don't market the scaffold as done just because it builds. What exists today? A terminal-UI dashboard. A session store backed by SQLite. Session start, stop, and resume flows. A background daemon mode. Observability and risk-scoring primitives. Worktree-aware session scaffolding. And basic multi-session state and output tracking. What's still missing? Richer multi-agent orchestration, explicit agent-to-agent delegation and summaries, a visual worktree and diff review surface, stronger external-harness compatibility, deeper memory and roadmap-aware planning, and a release and installer story. To try the alpha, you go into the control-plane directory and use Cargo to launch the dashboard, start a session with a task and an agent on a worktree, list sessions, inspect a session's status, stop a session, resume a stopped or failed one, run the daemon loop, and run the tests. Think of it as a preview of where ECC is heading: from configs for one assistant to an operations console for a fleet of agents.

You don't need the Rust alpha to get operator features, though. The Node-based operator command-line interface is stable and ships with the package. It exposes a few binaries. There's the main operator and lifecycle tool. There's the selective-install applier. And there's a control-pane substrate. For lifecycle and health, you can list what ECC installed, run a doctor command to diagnose problems, run a repair command to restore ECC-managed files, and run uninstall in dry-run mode. For discovery and install, you can run the consult command to find matching components, and the install command with profiles and capabilities. And there's a genuinely useful feature for teams and handoffs: operator status snapshots. You turn local state into a portable Markdown report with the status command. It covers readiness, active sessions, skill-run health, install health, pending governance events, and linked work items. Related commands let you add manual work-item entries, sync the pull-request and issue queue from GitHub, and fail automation when readiness needs attention.

ECC also includes tooling for running and supervising agent loops and parallel work. There are commands to start and inspect controlled agentic loops, with a loop-operator agent that monitors for stalls and intervenes. There's a persistent REPL with model routing, skill hot-loading, and session branching, search, export, compaction, and metrics. There's an autonomous-loops skill, with patterns for sequential pipelines, pull-request loops, and graph-based orchestration, including guards against runaway observer loops. There's a worktree orchestration script that uses a terminal multiplexer for multi-agent work. And there's a harness-audit command that scores your harness setup for reliability, eval readiness, and risk, with a harness-optimizer agent that tunes it. These are powerful and a little dangerous, so re-read the security chapter before running anything autonomously.

There are also a number of npm scripts worth knowing. There's a full validation command that checks unicode safety, validates agents, commands, rules, skills, and hooks, runs catalog and registry checks, and then runs the test suite. There's a command to launch the desktop interface. There's the harness reliability and risk scoring. There's an observability readiness check. There's an operator readiness dashboard. There's the control-pane substrate. There's a supply-chain scan. There's the persistent REPL. And there's an orchestration status command.

So, the key takeaways. ECC two-point-oh is the operator layer above individual installs: manage many sessions, see state and risk, and orchestrate. The Rust control plane is real but alpha, with dashboard, start, sessions, status, stop, resume, and daemon commands. The operator command-line interface is stable today, with consult, install, doctor, repair, list-installed, uninstall, and the Markdown status report. And the orchestration tooling exists, but use it carefully.
