第十六章。ECC 2.0 同操作者命令列介面。

到目前為止，我哋將 ECC 當做一個你安裝入一款框架嘅目錄。ECC 2.0 引入一個喺嗰個之上嘅層：一個由一個介面去操作好多代理會話嘅方式。呢一章講 Rust 控制平面原型，同你今日可以用嘅操作者命令列介面。

我哋由動機開始講。一旦你行緊幾個 agents，橫跨唔同嘅 worktrees、唔同嘅框架，有啲喺度自主回圈，你就需要操作者問題嘅答案。邊啲會話運行緊？邊啲卡住咗？邊啲需要審查？風險係乜？同埋我喺邊度停低咗？嗰個就係一個控制平面嘅工作。ECC 2.0 係個別框架安裝之上嗰一層：佢管理好多代理會話、令狀態、輸出同風險保持可見，並且加入編排同審查控制。佢係 Claude Code 優先，但唔阻礙未來嘅框架互通性。

當前嘅 ECC 2.0 鷹架係一個基於 Rust 嘅控制平面，而 repo 對佢嘅狀態好誠實，所以你都應該誠實。佢係真實嘅程式碼、alpha 品質、可以喺本機建構同測試。但佢唔係完成咗嘅 ECC 2.0 產品。Repo 甚至有一條 rule：唔好淨係因為佢建構得到，就將個鷹架當做完成咗咁去推銷。今日有啲乜存在？一個終端 UI 儀表板。一個由 SQLite 支援嘅會話儲存。會話開始、停止同恢復嘅流程。一個背景守護程序模式。可觀測性同風險評分嘅基礎元件。Worktree 感知嘅會話搭建。同基本嘅多會話狀態同輸出追蹤。仲欠啲乜？更豐富嘅多代理編排、明確嘅代理對代理委派同摘要、一個視覺化嘅 worktree 同差異審查介面、更強嘅外部框架相容性、更深嘅記憶同感知路線圖嘅規劃，同一個發佈同安裝器嘅故事。要試 alpha，你入去控制平面目錄，用 Cargo 去啟動儀表板、用一個任務同一個 agent 喺一個 worktree 上面開始一個會話、列出會話、檢查一個會話嘅狀態、停止一個會話、恢復一個停止咗或者失敗咗嘅、運行守護程序迴圈，同運行測試。將佢諗成 ECC 去緊邊度嘅一個預覽：由畀一個助手嘅設定，去到一個畀一支代理艦隊嘅操作主控台。

不過，你唔需要 Rust alpha 都攞到操作者功能。基於 Node 嘅操作者命令列介面係穩定嘅，而且隨套件附帶。佢暴露咗幾個二進位檔案。有主要嘅操作者同生命週期工具。有選擇性安裝嘅套用器。仲有一個控制面基底。至於生命週期同健康，你可以列出 ECC 裝咗啲乜、執行一個 doctor command 去診斷問題、執行一個 repair command 去還原 ECC 管理嘅檔案，同埋用試行模式執行解除安裝。至於發現同安裝，你可以執行 consult command 去搵相符嘅組件，同用設定檔同能力去執行 install command。仲有一個對團隊同交接真係有用嘅功能：操作者狀態快照。你用 status command 將本機狀態變成一份可攜帶嘅 Markdown 報告。佢涵蓋就緒度、活躍會話、skill 運行健康、安裝健康、待定嘅管治事件，同連結嘅工作項目。相關嘅 commands 等你加入手動嘅工作項目條目、由 GitHub 同步拉取請求同議題佇列，並且喺就緒度需要關注時令自動化失敗。

ECC 仲包含運行同監督代理回圈同並行工作嘅工具。有 commands 去開始同檢查受控嘅代理式回圈，配一個 loop-operator agent，佢監察卡頓並且介入。有一個持久嘅 REPL，有模型路由、skill 熱載入，同會話分支、搜尋、匯出、壓縮，同度量。有一個 autonomous-loops skill，有畀依次序流水線、拉取請求回圈，同基於圖嘅編排嘅模式，包括防止失控觀察者回圈嘅守衛。有一個 worktree 編排腳本，用一個終端機多工器嚟做多代理工作。仲有一個 harness-audit command，為你嘅框架設置評分可靠性、評估就緒度同風險，配一個 harness-optimizer agent 去調校佢。呢啲嘢強大但有少少危險，所以喺自主運行任何嘢之前，重新睇一次安全嗰章。

仲有一啲值得知嘅 npm 腳本。有一個完整驗證 command，會檢查 unicode 安全、驗證 agents、commands、rules、skills 同 hooks、運行目錄同登記檢查，然後運行測試套件。有一個 command 去啟動桌面介面。有框架可靠性同風險評分。有一個可觀測性就緒度檢查。有一個操作者就緒度儀表板。有控制面基底。有一個供應鏈掃描。有持久嘅 REPL。仲有一個編排狀態 command。

咁，重點精華。ECC 2.0 係個別安裝之上嘅操作者層：管理好多會話、睇狀態同風險，並且編排。Rust 控制平面係真實但 alpha，有 dashboard、start、sessions、status、stop、resume，同 daemon commands。操作者命令列介面今日係穩定嘅，有 consult、install、doctor、repair、list-installed、uninstall，同 Markdown 狀態報告。而編排工具存在，但要小心用。
