Chapter nineteen. Glossary and quick reference.

This is the one-page cheat sheet, the chapter to bookmark. Let's run through the key terms.

ECC is Everything Claude Code, the agent-harness operating system at the center of this book. It is not elliptic curve cryptography. A harness is the program wrapping a model that lets it act, like Claude Code, Cursor, Codex, or OpenCode. An agent, or sub-agent, is a scoped specialist the orchestrator delegates to, with its own context, limited tools, and a chosen model. The orchestrator is the main assistant that delegates to agents and assembles their results. A skill is the canonical reusable workflow unit. A command is a typed slash entry, and it's a compatibility layer over skills. A hook is an automation triggered by a tool or session event, mapping a configuration file to small scripts. A rule is an always-follow guideline loaded every session, both common ones and per-language ones. MCP, the Model Context Protocol, is a server exposing tools the model can call, like GitHub, a database, or a browser. An instinct is a small, confidence-scored unit of learned behavior in the second-generation learning system. A codemap is a lightweight map of a codebase so the assistant navigates without re-exploring. A profile is an install bundle, either minimal, core, or full. A target is the harness an install is for. The agent data home is the root for ECC's session, memory, and metrics data. The lethal trifecta is private data plus untrusted content plus external communication in one runtime, which equals exfiltration risk. AgentShield is ECC's bundled security auditor. The prompt-defense baseline is the anti-injection rule block present in every agent and in the main instructions file. ECC two-point-oh is the Rust control-plane prototype for managing many sessions, still in alpha. And pass-at-k versus pass-to-the-k are eval reliability metrics, meaning at least one of k attempts succeeds, versus all k attempts succeed.

Remember the three names of ECC. The GitHub repo uses one name. The plugin and marketplace use a short identifier. And the npm package uses a third name. They all refer to the same project.

Here's the install quick reference, in plain terms. For the plugin, inside Claude Code, you add the marketplace and install the plugin. For the manual route, you clone the repo, install dependencies, and run the installer with a profile and target; minimal means no hooks, you can keep core while disabling hooks, you can add hooks later with the hooks-runtime module, and you can install without cloning using the npx installer. To find what to install, use the consult command. For rules, which are always manual, make a rules directory and copy the common rules plus your stack's language pack. For lifecycle and reset, use list-installed, doctor, and repair, then the dry-run uninstall, then the real uninstall. And for other harnesses, you target Cursor, run the sync script for Codex, use the minimal profile for Zed, or just launch OpenCode in the repo. The golden rule, again: pick one path. Never the plugin and the full installer together.

The commands also group by job. There's a plan-and-build group, a testing group, a review group, a build-fix group, a verify group, a sessions group, a learning group, a docs group, a loops group, a harness group, and a pull-request group. The plugin form prefixes commands with the ecc namespace.

There's also an environment-variable reference worth knowing. The hook-profile variable sets strictness to minimal, standard, or strict. The disabled-hooks variable takes a comma-separated list of hook ids to turn off. There are variables to cap or disable the session-start context, to set the session retention window, and to hide the cost estimates. The agent-data-home variable sets the root for session, memory, and metrics data, which is how you isolate harnesses. The disabled-MCPs variable is an install and sync filter to skip bundled servers. There's a variable to enable governance event capture, one for where the second-generation learning system stores its instincts, one to force a specific package manager, and the base-URL and auth-token variables for a custom gateway, where you should trust only your own.

For token-optimization defaults, in your settings use the balanced model, set max-thinking-tokens to ten thousand, and set auto-compaction to fifty percent. Route models by task: cheapest for search and simple work, balanced as the default for coding, and most powerful for architecture, security, and hard debugging. Keep fewer than ten MCPs and fewer than eighty tools. Use semantic search. And keep files modular.

For the security minimum bar: use a separate identity with scoped tokens; isolate untrusted work in a container or virtual machine with no network; set deny rules for your SSH keys, cloud credentials, environment files, and dangerous shell commands; scope tools per workflow with least privilege; sanitize input and keep the prompt-defense baseline; require approvals, with logging and a kill switch; run the AgentShield scan and fix the criticals; and don't clone and open untrusted repos on your host.

And here is the mental model in one picture. At the top is identity and governance, the constitution. It governs the component catalog, made of agents, skills, commands, rules, and MCP. That catalog is shaped by the adapters and installer, with their manifests, state store, and target flag. And that lands and is enforced by the runtime, made of hooks and scripts, memory, learning, metrics, and the ECC two-point-oh control plane. Five principles cut across all layers: agent-first, test-driven, security-first, immutability, and plan-before-execute.

Check the command quick-reference for the live command list. And explore the docs folder for deep dives into architecture, releases, per-harness guides, and internationalization.

And with that, you've reached the end. You now know what ECC is, why it exists, how it's built, how to install it cleanly, how to operate it day to day across harnesses, how to make it learn, how to keep it fast, and how to keep it safe. Go build, and let the skills compound.
