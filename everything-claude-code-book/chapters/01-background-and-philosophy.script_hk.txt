第一章。背景同理念。

我哋就由 ECC 嘅由來開始講。ECC 係 Affaan Mustafa 嘅作品，佢由 Claude Code 喺實驗階段推出嗰陣開始就一直喺度用。呢個專案係由真實嘅產品工程裏面成長出嚟，最矚目嘅就係佢做咗 zenith dot chat，呢個產品贏咗 Anthropic 同 Forum Ventures 嘅黑客松。Repo 自己對佢嘅起源講得好直白。佢話佢唔淨止係一堆設定，而係一套完整嘅系統，喺超過十個月密集嘅每日使用裏面、邊起真實產品邊演化出嚟。

呢個細節好重要。ECC 唔係一份理論上嘅最佳實踐範本，唔係邊個喺一個周末就寫得出嚟。佢係幾千個鐘代理式編程嘅化石紀錄。每一個 hook、每一條 rule、每一個 skill 嘅存在，都係因為某個真實嘅失敗模式令到佢變得必要。當你讀到一條好似「永遠唔好 commit 一句除錯日誌」嘅 rule，或者一個會阻止開發伺服器喺終端機多工器以外運行嘅 hook，你其實係喺度讀一道疤痕。

個名係一個變成大專案嘅小笑話：Everything Claude Code。佢一開始以 Claude 為中心，然後擴展到涵蓋成個 AI 編程工具嘅生態系統。

呢度有一條由更新日誌整理出嚟嘅簡短時間線。你唔使背佢，但佢顯示咗發展嘅方向。1.2 版統一咗 commands 同 skills，加入咗會話管理，並且引入咗用 instincts 嘅持續學習。1.3 版帶嚟咗完整嘅 OpenCode 外掛支援。1.4 版加入咗多語言 rules、一個安裝精靈，同多代理 commands。1.6 版加入咗 Codex 命令列支援、AgentShield 安全審計器，同一個 GitHub Marketplace 應用程式。1.7 版跨平台擴展，並且加入咗一個簡報製作 skill。1.8 版將成件事重新定位做一套框架效能系統，有 hook 可靠性大改造同運行時 hook 控制。1.9 版引入咗選擇性安裝、更多語言支援，同一個 SQLite 狀態儲存。而 2.0 版就變成咗代理框架操作系統，有控制平面基底、worktree 生命週期、一個 orchestrator 家族，同一個 Discord 社群。主軸好清楚：ECC 由一個設定包，變成咗一套用嚟運行代理嘅操作系統。

咁究竟乜嘢係代理框架，又點解要優化佢呢？框架係一個包住語言模型嘅程式，令佢真正可以做到嘢：讀取檔案、執行 shell 指令、編輯程式碼、呼叫工具。Claude Code、Cursor、Codex 同 OpenCode 全部都係框架。模型係引擎。框架就係包住佢嘅架車。

呢度係 ECC 建基於上面嘅關鍵洞見。代理式編程嘅品質、成本同安全，大部分係由框架決定，而唔係由模型決定。兩個人用完全相同嘅模型，得出嚟嘅結果可以差天共地，視乎幾樣嘢。模型睇到啲乜上下文，以及佢嘅視窗有幾多被浪費咗。邊啲子任務被委派畀更平、或者更專門嘅配置。有啲乜嘢被自動檢查，例如測試、格式化同秘密資料，相對於有啲乜嘢就只能靠彩數。模型喺會話之間記得啲乜。以及模型被容許做啲乜，亦即係工具權限同沙箱。

ECC 係一套框架效能系統：一套經過精心挑選、又經得起實戰考驗嘅配置，可以一次過調校以上所有嘅槓桿。呢度有一個好用嘅比喻。模型係一位出色嘅新員工。框架係佢嘅手提電腦同帳號。ECC 就係入職手冊、一隊導師、測試流水線，同安全政策。佢係將原始天賦變成可靠產出嘅一切。

而家我哋講吓五大核心原則。Repo 根目錄有三個細細嘅檔案，編碼咗 ECC 嘅世界觀：一個講身份，一個講硬性約束，仲有助手會讀嘅指令。有五個原則喺到處反覆出現。

原則一：代理優先。盡早將工作分派畀啱嘅專家。唔係一個通才咩都做，ECC 將工作推畀有明確範圍嘅子代理。一個 planner 負責策略，一個 code reviewer 負責品質，一個 security reviewer 負責敏感程式碼，一個 build-error resolver 喺工具鏈壞咗嗰陣出動。咁就令到每一個上下文都細，每一個代理都係專家。

原則二：測試驅動。喺信任實作改動之前，先寫或者更新測試。ECC 規定要行紅、綠、重構嘅迴圈，同埋喺單元、整合同端對端測試上面，達到八成以上嘅覆蓋率目標。呢個係由一個測試驅動開發 skill，同埋一啲推你去達標覆蓋率嘅 hooks 一齊強制執行。

原則三：安全優先。驗證輸入、保護秘密資料，同保持安全嘅預設值。每一個代理都帶住一條提示防禦基線。Hooks 會掃描秘密資料。有成份安全指南同一個專門嘅掃描器，我哋會用成個章節去講佢。

原則四：不可變性。寧願用明確嘅狀態轉換，都唔好用變更。呢個係一條編程風格原則：建立新嘅物件，而唔係變更現有嘅物件。佢出現喺 rules 入面、reviewer 代理嘅清單入面，同埋通篇嘅範例程式碼入面。

原則五：先規劃，後執行。複雜嘅改動應該拆做一個個刻意安排嘅階段。plan command 同 planner 代理嘅存在，就係要阻止助手喺冇藍圖嘅情況下，貿貿然衝入一個改十二個檔案嘅改動。先規劃，攞到確認，然後先至郁程式碼。

呢五條就係憲法。如果你幾時諗唔通點解 ECC 用某種方式做事，答案幾乎一定追溯返到其中一條。

仲有一套複利工作流程嘅理念。長篇指南引用咗一句話，捕捉到 ECC 最深層嘅賭注。早期，作者花咗時間建立可重用嘅工作流程同模式。建立佢嗰陣好沉悶，但隨住模型同代理框架改進，佢產生咗一種誇張嘅複利效應。論點係咁：你今日打嘅提示用完即棄。但你今日寫嘅 skill 會持續帶嚟回報，每一次會話、每一個專案，而且隨住底層模型改進，佢會變得越嚟越好。所以 ECC 大力投資喺耐用嘅產物上面：可重用嘅專家、可重用嘅工作流程、規劃模式，同上下文工程模式，好似記憶、壓縮，同系統提示注入。呢個就係點解技能優先係一句不斷重複嘅口號。一段用完即棄嘅對話係一支蠟燭。一個 skill 係一個電源插座。

咁你實際上會得到啲乜，用數字嚟講呢？喺 2.0 版，個表面係咁，記住啲數目每個禮拜都會變，所以當佢係一張快照。64 個 agents，係專門做規劃、審查、修復建構，同特定語言工作嘅子代理。262 個 skills，涵蓋工作流程同領域知識。84 個 commands，係斜線項目，再加上一個可選擇加入嘅、已退役墊片嘅封存區。Hooks，係橫跨會話生命週期嘅自動化。Rules，涵蓋通用標準，再加上 TypeScript、Python、Go、Swift、PHP、ArkTS 等等嘅各語言包。MCP 設定，係連接外部服務嘅連接器定義。跨平台腳本，用嚟安裝、hooks、會話同編排。一個 Rust 控制平面原型，係 ECC 2.0 嘅早期版本。仲有一個桌面圖形介面，畀你瀏覽以上一切。

唔好畀啲數目嚇親。九成時間你只會用到一手指頭數得晒嘅 agents，同十幾個 skills。其餘嘅係喺有冷門需求嘅時候先出現，無論係 Kotlin 建構錯誤、HarmonyOS 應用程式、預測市場研究，定係連影片剪輯都得。係，真係得。

咁，重點精華。ECC 即係 Everything Claude Code，一套由超過十個月真實每日使用孕育出嚟嘅框架效能系統。框架包住模型，令佢可以行動，而 ECC 調校嘅係框架，唔係模型。五個原則貫穿一切：代理優先、測試驅動、安全優先、不可變性，同先規劃後執行。ECC 押注喺耐用、有複利效應嘅工作流程上面，而唔係用完即棄嘅提示。而啲大數目係一個自助餐。你只係夾你需要嘅嘢。
