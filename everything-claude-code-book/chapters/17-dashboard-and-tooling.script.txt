Chapter seventeen. Dashboard and ecosystem tooling.

ECC is mostly a terminal experience, but it also ships a graphical front door and a small ecosystem of companion tools. This short chapter covers the friendly surfaces.

Let's start with the desktop dashboard. There's a desktop graphical interface, built with the Tkinter toolkit, for visually exploring everything in the catalog. You launch it through an npm script, or by running the Python file directly. Its features include a tabbed interface for agents, skills, commands, rules, and settings; search and filter across all components; a dark and light theme toggle; font customization for family and size; and the project logo in the header and taskbar. It's the easiest way to browse sixty-four agents and two hundred and sixty-two skills without printing out Markdown files, and a nice way for a new teammate to get oriented. It requires Python 3 with the Tkinter toolkit available. If the npm script can't find it, you run the Python file directly and confirm Tkinter is installed for your Python.

Next, the skill creator. There are two ways to generate skills from a repository, which we also touched on earlier. The local, built-in way analyzes your git history locally and writes skill files, with an option to also seed the second-generation learning system, and no external service. The advanced way is a GitHub App for very large repos, automatic pull requests, and team sharing, through a hosted marketplace app. Both produce ready-to-use skills, instinct collections, and pattern extraction from your commit history.

There's AgentShield, which we covered in depth in the security chapter. The one-liner: it's a security auditor for your agent config, runnable with a single command, with a deep red-team, blue-team, and auditor mode, and a continuous-integration-friendly exit code. Inside the harness, you run it as a security scan.

There's also a cluster of skill and harness health tools that keep your setup honest over time. There's a skill-health dashboard with analytics. There's a skill-stocktake to audit skills and commands for quality. There's a harness-audit to score harness reliability, eval readiness, and risk. And there's a rules-distill command to extract cross-cutting principles from skills and turn them into rules. There are command-line equivalents for the harness audit, an observability readiness check, and an operator readiness dashboard. These are how you prevent skill sprawl and config drift as your catalog grows.

On package-manager setup: ECC auto-detects your package manager, whether that's npm, pnpm, yarn, or bun, in a priority order that starts with an environment variable, then a project config file, then the package-manager field in your project file, then a lock file, then global config, and finally the first one available. You can also set it explicitly, with an environment variable, a global or project setup command, a detect command, or an in-harness setup command.

There's a productivity note about editor pairing, because the editor shapes the experience. The shortform guide is opinionated here. Its top pick is Zed, which is Rust-fast, tracks file changes in real time through its agent panel, has a quick command palette, and uses low resources, which matters when running the most powerful model. VS Code and Cursor also work well, using the terminal with the IDE integration for the language server, or the integrated extension. Editor-agnostic tips: split your screen between the assistant and the editor, enable autosave so file reads are current, use git integration to review diffs before committing, and confirm file watchers auto-reload changed files.

Here's the ecosystem at a glance. The desktop dashboard is a graphical interface to browse the catalog. The skill creator generates skills from git history. AgentShield is the security auditor. The hosted GitHub App offers hosted analysis, pull-request audits, and private-repo support. The persistent REPL gives you routing and metrics. The operator command-line interface handles operator and lifecycle work. And the Rust control plane is the alpha for ECC two-point-oh. The open-source repo is MIT-licensed and free forever. The paid pieces, the hosted GitHub App and sponsorships, fund the work, which is how a single maintainer ships weekly across many harnesses. You never need the paid pieces to use everything in this book.

So, the key takeaways. The dashboard script opens a graphical interface to browse agents, skills, commands, and rules. The skill-create command generates skills from your git history locally, with no service needed. You keep your setup healthy with the skill-health, skill-stocktake, harness-audit, and rules-distill tools. You set your package manager through the setup command or an environment variable. And the open-source core is free, while the hosted tools are optional.
