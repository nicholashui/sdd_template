第十三章。持續學習。

我哋由佢解決嘅問題開始講。你哋全部都試過呢樣嘢：你連續三日為同一樣嘢糾正個助手。佢撞到同一個 bug，你畀同一個變通做法佢，佢又唔記得，然後你又再嚟過。咁就浪費咗 token、浪費咗上下文，亦浪費咗時間。ECC 嘅答案係咁：當個助手發現咗一啲唔簡單嘅嘢，一個除錯技巧、一個專案特有嘅怪癖、一個變通做法，佢就應該將嗰樣嘢存做可重用嘅知識，咁下次個模式就會自動載入。個目錄係由你自己嘅使用成長出嚟。

值得提一提，唔好將記憶同學習混淆，因為佢哋唔同。記憶，即係之前嗰章講嘅，係呢個專案喺會話之間攜帶嘅狀態。持續學習就係關於耐用嘅模式，被抽取成 instincts 同 skills，並且套用喺跨會話同跨專案。

ECC 內附咗呢樣嘢嘅兩代，而你應該知道你用緊邊一代。第一代用一個停止 hook，將模式抽取成學到嘅 skills。佢簡單而且經過驗證。第二代係基於 instinct，有信心評分，再加引入、匯出，同演化。淨係喺你明確咁想要嗰個舊版學到 skill 嘅流程時，先保留第一代。否則，偏向用第二代。第二代系統係推薦嘅路線。

我哋講吓 instincts，即係學習嘅單位。一個 instinct 係一小塊有評分嘅學到行為：一個觀察加一個動作，連同一個信心評分同支持嘅證據同範例。Instincts 喺你工作嗰陣靜靜咁累積，因為觀察 hooks 捕捉工具使用嘅訊號，而你用 commands 嚟管理佢哋。有一個 instinct-status command，去顯示你學到嘅 instincts 同佢哋嘅信心評分，包括專案層級同全域。有一個 instinct-import command，去由一個檔案、一個 URL，或者一個隊友引入 instincts。有一個 instinct-export command，去分享你嗰啲。有一個 prune command，去刪除已過期嘅待定 instincts，佢哋有一個三十日嘅存活時間。有一個 promote command，去將專案範圍嘅 instincts 提升到全域範圍。仲有一個 projects command，去列出已知嘅專案同佢哋嘅 instinct 統計。Instincts 係有範圍嘅：一個喺一個專案學到嘅 instinct，喺你將佢提升到全域之前，會保持喺專案範圍。咁就阻止咗專案嘅怪癖洩漏到處都係，同時仍然等真正普遍嘅教訓畢業。順帶講一講內部運作，instinct 儲存同會話記憶係分開嘅，所以清除你嘅會話，唔會抹走你學到嘅嘢。

個別嘅 instincts 有用，但真正嘅魔法，係將相關嘅 instincts 聚集成一個全新嘅 skill，而呢個就係 evolve command 做嘅嘢。個流程係咁：一個 learn command 由當前會話抽取可重用嘅模式，一個 learn-and-evaluate command 抽取模式並且喺儲存之前自我評估佢哋嘅品質，而 evolve command 分析你學到嘅 instincts 並且建議演化咗嘅 skill 結構。由頭到尾，個迴圈係：一個重複嘅模式被 hooks 觀察到，佢被存做一個帶信心評分嘅 instinct，evolve command 將相關嘅 instincts 聚集成一個新 skill，而嗰個新 skill 喺下次情況相符時自動載入。呢個就係一個通用嘅助手點樣變成你嘅助手：你不斷教佢嘅嘢，凝固成永遠都喺度嘅 skills。

有一條互補嘅路線，唔使等 instincts 累積：由你嘅 git 歷史生成 skills。你執行 skill-create command，去分析當前 repo 嘅 commit 歷史並且產出 skill 檔案，而你可以加一個旗標，順便為第二代學習系統生成 instincts。佢喺本機讀取你嘅 commit 歷史，並且將模式抽取成即用嘅 skills。對於超大型 repo、團隊分享，或者自動拉取請求，有一條進階嘅 GitHub App 路線，但 skill-create command 喺冇外部服務嘅情況下涵蓋咗常見嘅情況。你亦都可以審計你累積知識嘅健康狀況：有一個帶分析嘅 skill-health 儀表板、一個 skill-stocktake 去審計 skills 同 commands 嘅品質，同一個 rules-distill command，會掃描 skills、抽取橫跨各處嘅原則，並且將佢哋提煉成 rules。

值得重複講一講，點解捕捉喺停止事件運行，而唔係喺每一個提示。呢個係一個深思熟慮嘅設計選擇。學習捕捉喺會話結束時運行一次，而唔係喺每一個訊息，因為逐個訊息運行佢，會為每一個提示加上延遲。停止運行一次，輕量，唔阻你手腳。觀察 hooks 確實會喺會話期間捕捉輕量嘅訊號，係非同步而且有一個短逾時，但沉重嘅抽取係喺結尾發生。

你唔使第一日就做晒呢啲。呢度有一個合理嘅採用斜坡。第一個禮拜，淨係工作。等 instincts 累積，偶爾執行 instinct-status 去睇吓佢留意緊乜。第二個禮拜，喺紮實嘅會話結尾，執行 learn-and-evaluate command，然後儲存個會話。第三個禮拜，執行 evolve command 去將一群群 instincts 變成 skills，並且執行 skill-health 去檢查品質。而持續咁，將真正普遍嘅教訓提升到全域、匯出 instincts 同隊友分享，並且 prune 去保持企理。

咁，重點精華。持續學習將重複嘅糾正，變成耐用、自動載入嘅知識。偏向用第二代系統，有 instincts 同信心評分，多過第一代嘅停止 hook 流程。用 status、import、export、prune 同 promote commands 嚟管理 instincts，並且記住 instincts 喺被提升之前係專案範圍。Evolve command 將 instincts 聚集成新 skills，而 skill-create command 開採你嘅 git 歷史。而捕捉喺停止 hook 運行，去避免逐個提示嘅延遲。
