第十二章。跨框架使用。

呢度係大概念：寫一次，到處運行。雖然個名叫 Everything Claude Code，但 ECC 係多工具嘅。佢自稱係第一個將每一款主要 AI 編程工具發揮到極致嘅外掛。你編寫你嘅 agents、skills 同 rules 一次，而安裝器就將佢哋翻譯成每一款框架嘅原生佈局。

我哋睇吓框架計分卡，因為功能涵蓋因工具而異，而呢啲數目係一張會變嘅快照。Claude Code 係最豐富嘅，有 64 個 agents、84 個 commands、262 個 skills、八個 hook 事件、34 條 rules、十四個 MCP 連接器，同佢自己嘅上下文檔案。Cursor 共用 agents、commands 同 skills，有更多嘅 hook 事件，達到十五個，用佢自己嘅前言區格式攜帶 rules，並且用通用上下文檔案。Codex，無論係佢嘅應用程式定命令列形式，都共用 agents，並且對 commands 同 rules 用一個基於指令嘅做法，暫時冇 hook 事件，並且用一個設定檔嚟處理佢嗰一手指頭數得晒嘅 MCP 伺服器。OpenCode 有佢自己嘅原生一套：十二個 agents、35 個 commands、37 個 skills、十一個 hook 事件、十三條 rules，同完整嘅 MCP 支援。而 GitHub Copilot 係最輕量嘅，有一手指頭數得晒嘅提示、基於指令嘅 skills 同 rules、冇 hook 事件，同一個成日存在嘅單一指令檔案。可以咁讀：Claude Code 係參考目標，有最豐富嘅介面。Cursor 同 OpenCode 其實有更多嘅 hook 事件。而 Codex 同 Copilot 冇一個 hook 系統，所以嗰度嘅強制執行係基於指令，而唔係運行時。

Claude Code 係主要嘅原生目標。本書入面所有嘢都直接適用。你透過外掛或者手動安裝器去安裝，而佢需要命令列工具係 2.1.0 版或者之後。

至於 Cursor，預先翻譯好嘅設定住喺佢嘅設定目錄，而你透過將目標設做 Cursor 同你嘅語言去安裝。你會得到：十五個 hook 事件、十六個薄薄嘅 hook 腳本，透過一個轉接器委派畀共用腳本、用 Cursor 前言區格式嘅 34 條 rules、用一個前綴安裝、避免碰撞嘅 agents，再加共用嘅 skills、commands 同 MCP。記憶隔離喺呢度好重要。因為 Cursor 重用同樣嘅 hook 腳本，ECC 自動將 Cursor 嘅記憶隔離喺你嘅 Claude 目錄之外，用一個設定獨立資料家目錄嘅會話開始 hook，連同一個預設位置、一個覆寫檔案，同一條成日生效嘅 rule。如果你刻意想同 Claude Code 共用記憶，你就將資料家目錄變數指向你嘅 Claude 目錄。Cursor 嘅 hook 系統其實比 Claude 嘅更豐富，有好似 shell 執行前、檔案編輯後、用嚟偵測秘密資料嘅提交提示前、分頁檔案讀取前嘅事件，後者會阻止分頁功能讀取好似環境同金鑰檔案咁嘅敏感檔案，仲有 MCP 執行前後嘅事件用嚟做審計記錄。

至於 Codex，即係 macOS 應用程式同命令列工具，透過通用上下文檔案加一個設定檔，有一等嘅支援。最乾淨嘅設置係同步腳本，佢會對你嘅 Codex 設定做一個只加入嘅合併，有一個預覽選項，同一個重新整理伺服器嘅旗標。又或者你可以淨係喺 repo 入面行 Codex，因為個上下文檔案同設定目錄會自動偵測。參考設定刻意唔釘死個模型或者供應商，因為 Codex 用佢自己嘅預設。包括咗啲乜：頂層嘅批准、沙箱同網絡搜尋設定、六個 MCP 伺服器，或者加咗一個資料庫就七個、32 個 skills、兩個設定檔，一個嚴格唯讀嘅同一個全自動批准嘅，同三個範例 agent 角色，一個探索員、一個 reviewer，同一個文件研究員。關鍵限制：Codex 暫時冇 hook 執行嘅對等，所以 ECC 嘅強制執行係基於指令，透過個上下文檔案、可選嘅指令覆寫，同沙箱同批准設定。實驗性嘅 Codex 外掛 marketplace 路線，喺上游好脆弱，所以偏向用同步腳本。

仲有一條長尾嘅其他工具。有畀 Gemini 命令列工具嘅實驗性專案本地支援。有一個保守嘅 Zed 轉接器，喺嗰度你將 API 金鑰放喺 Zed 自己嘅設定，而唔係 repo。有畀 Antigravity 嘅緊密整合設置。有畀 JoyCode 同 CodeBuddy 嘅專案本地選擇性安裝轉接器。有畀 Qwen 命令列工具嘅家目錄轉接器。而對於好似 Grok 嘅非原生工具，有一條手動後備路線。呢啲每一個喺文件入面都有佢自己嘅指南。

你亦都可以喺一個自訂端點或者閘道上面行 ECC。ECC 唔硬編碼 Anthropic 嘅傳輸，所以佢透過 Claude Code 正常嘅閘道支援運作。你設定一個 base-URL 環境變數同一個 auth-token 變數，然後啟動。如果你嘅閘道重新對應模型名，你喺 Claude Code 入面設定嗰樣嘢，而唔係喺 ECC，因為一旦個助手行得到，ECC 嘅 skills、hooks、commands 同 rules 就同供應商無關。一個警告：一個惡意專案覆寫嗰個 base-URL 變數，曾經係一個真實世界嘅漏洞，我哋會喺安全嗰章講。
