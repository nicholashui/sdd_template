第十九章。词汇表和快速参考。

这是一页式的速查表，是值得收藏的一章。我们过一过那些关键术语。

ECC 也就是 Everything Claude Code，本书中心那套代理框架操作系统。它不是椭圆曲线密码学。一个框架是包住一个模型、让它可以行动的程序，例如 Claude Code、Cursor、Codex，或者 OpenCode。一个 agent，或者说 sub-agent，是 orchestrator 委派工作给它的、有明确范围的专家，有它自己的上下文、有限的工具，和一个挑好的模型。Orchestrator 是委派工作给 agents 并且汇集它们结果的主助手。一个 skill 是标准的可重用工作流程单位。一个 command 是一个打出来的斜杠项目，而它是 skills 之上的一个兼容层。一个 hook 是由一个工具或者会话事件触发的自动化，把一个配置文件对应到小小的脚本。一条 rule 是一个每一次会话都会加载、要永远遵守的指引，有通用的也有各语言的。MCP，也就是 Model Context Protocol，是一个暴露出模型可以调用的工具的服务器，例如 GitHub、一个数据库，或者一个浏览器。一个 instinct 是第二代学习系统里面，一小块有信心评分的学到行为单位。一个 codemap 是一份代码库的轻量地图，让助手不用重新探索地去导航。一个 profile 是一个安装套装，可以是 minimal、core，或者 full。一个 target 是一个安装是给哪款框架的。代理数据主目录是 ECC 的会话、记忆和度量数据的根目录。致命三重奏是私人数据加不可信内容加对外通讯在一个运行时里面，等于外泄风险。AgentShield 是 ECC 内置的安全审计器。提示防御基线是出现在每一个 agent 和主指令文件里面那个反注入的 rule 块。ECC 2.0 是管理很多会话的 Rust 控制平面原型，仍然在 alpha。而 pass-at-k 对 pass-to-the-k，是评估可靠性的度量，分别意味着 k 次尝试之中至少一次成功，对比 k 次尝试全部成功。

记住 ECC 的三个名字。GitHub repo 用一个名。插件和 marketplace 用一个简短的标识符。而 npm 包用第三个名。它们全部指的都是同一个项目。

这里是安装的快速参考，用浅白的话讲。至于插件，在 Claude Code 里面，你加入 marketplace 并且安装插件。至于手动路线，你 clone 那个 repo、安装依赖，并且用一个配置文件和目标执行安装器；minimal 意味着没有 hooks，你可以保留 core 的同时停用 hooks，你可以稍后用 hooks-runtime 模块加 hooks，而你可以用 npx 安装器、不用 clone 就安装。要找装什么，用 consult command。至于 rules，它们永远是手动的，建一个 rules 目录并且复制 common 的 rules 加你技术栈的语言包。至于生命周期和重设，用 list-installed、doctor 和 repair，接着试运行卸载，然后真正卸载。而至于其他框架，你把目标设成 Cursor、为 Codex 执行同步脚本、为 Zed 用 minimal 配置文件，或者只在 repo 里面启动 OpenCode。黄金法则，再讲一次：挑一条路线。永远不要插件和完整安装器一起用。


那些 commands 也按工作分组。有一个规划和构建的组、一个测试的组、一个审查的组、一个构建修复的组、一个验证的组、一个会话的组、一个学习的组、一个文档的组、一个循环的组、一个框架的组，和一个拉取请求的组。插件形式会在 commands 前面加一个 ecc 命名空间。

还有一份值得知道的环境变量参考。Hook 配置变量把严格度设成 minimal、standard，或者 strict。停用 hooks 的变量，接受一个以逗号分隔、要关闭的 hook id 清单。有变量去封顶或者停用会话开始的上下文、去设置会话保留窗口，和去隐藏成本估算。代理数据主目录变量设置会话、记忆和度量数据的根目录，这个就是你隔离各框架的方法。停用 MCP 的变量是一个安装和同步的过滤器，去跳过内置的服务器。有一个变量去启用治理事件捕捉、一个给第二代学习系统存放它的 instincts、一个去强制一个特定的包管理器，和 base-URL 和 auth-token 变量给一个自定义网关，在那里你应该只信任你自己那个。

至于 token 优化的默认，在你的设置，用平衡的模型、把最大思考 token 设成一万，并且把自动压缩设成五成。按任务路由模型：搜索和简单工作用最便宜的、平衡的做编程的默认，而最强大的就给架构、安全，和困难的调试。保持少于十个 MCP 和少于八十个工具。用语义搜索。并且保持文件模块化。

至于安全的最低门槛：用一个配有范围 tokens 的独立身份；在一个没有网络的容器或者虚拟机里面隔离不可信的工作；为你的 SSH 密钥、云端凭证、环境文件，和危险的 shell 命令设置拒绝 rules；用最小权限逐个工作流程限定工具；净化输入并且保留提示防御基线；要求批准，配记录和一个紧急停止开关；运行 AgentShield 扫描并且修了那些关键问题；和不要在你的主机上 clone 和打开不可信的 repos。

而这里是用一幅图讲的心智模型。在最顶是身份和治理，也就是宪法。它治理组件目录，由 agents、skills、commands、rules 和 MCP 组成。那个目录由适配器和安装器塑造，配它们的清单、状态存储，和目标标志。而那样东西由运行时落地和强制执行，运行时由 hooks 和脚本、记忆、学习、度量，和 ECC 2.0 控制平面组成。五个原则横切所有层：代理优先、测试驱动、安全优先、不可变性，和先规划后执行。

你下一步应该去哪？读 repo 自己的指南，从短篇指南开始，然后长篇指南，然后安全指南。看 command 快速参考，拿实时的 command 清单。并且探索 docs 文件夹，深入了解架构、发布、各框架指南，和国际化。

而讲到这里，你就到了结尾。你现在知道 ECC 是什么、它为什么存在、它怎样构建、怎样干净地安装它、怎样跨框架日常操作它、怎样让它学习、怎样让它保持快速，和怎样让它保持安全。去建造吧，并且让那些 skills 产生复利。
