第十六章。ECC 2.0 和操作者命令行界面。

到目前为止，我们把 ECC 当成一个你安装进一款框架的目录。ECC 2.0 引入一个在那之上的层：一个由一个界面去操作很多代理会话的方式。这一章讲 Rust 控制平面原型，和你今天可以用的操作者命令行界面。

我们从动机开始讲。一旦你在跑几个 agents，横跨不同的 worktrees、不同的框架，有些在自主循环，你就需要操作者问题的答案。哪些会话在运行？哪些卡住了？哪些需要审查？风险是什么？以及我在哪里停下了？那就是一个控制平面的工作。ECC 2.0 是单个框架安装之上的那一层：它管理很多代理会话、让状态、输出和风险保持可见，并且加入编排和审查控制。它是 Claude Code 优先，但不阻碍未来的框架互通性。

当前的 ECC 2.0 脚手架是一个基于 Rust 的控制平面，而 repo 对它的状态很诚实，所以你也应该诚实。它是真实的代码、alpha 质量、可以在本机构建和测试。但它不是完成了的 ECC 2.0 产品。Repo 甚至有一条 rule：不要只因为它能构建，就把脚手架当成完成了那样去推销。今天有什么存在？一个终端 UI 仪表板。一个由 SQLite 支持的会话存储。会话开始、停止和恢复的流程。一个后台守护进程模式。可观测性和风险评分的基础组件。Worktree 感知的会话搭建。和基本的多会话状态和输出追踪。还缺什么？更丰富的多代理编排、明确的代理对代理委派和摘要、一个可视化的 worktree 和差异审查界面、更强的外部框架兼容性、更深的记忆和感知路线图的规划，和一个发布和安装器的故事。要试 alpha，你进去控制平面目录，用 Cargo 去启动仪表板、用一个任务和一个 agent 在一个 worktree 上开始一个会话、列出会话、检查一个会话的状态、停止一个会话、恢复一个停止了或者失败了的、运行守护进程循环，和运行测试。把它想成 ECC 要去哪的一个预览：从给一个助手的配置，到一个给一支代理舰队的操作主控台。


不过，你不需要 Rust alpha 都能得到操作者功能。基于 Node 的操作者命令行界面是稳定的，而且随软件包附带。它暴露了几个二进制文件。有主要的操作者和生命周期工具。有选择性安装的应用器。还有一个控制面底座。至于生命周期和健康，你可以列出 ECC 装了什么、执行一个 doctor command 去诊断问题、执行一个 repair command 去还原 ECC 管理的文件，以及用试运行模式执行卸载。至于发现和安装，你可以执行 consult command 去找相符的组件，和用配置文件和能力去执行 install command。还有一个对团队和交接真的有用的功能：操作者状态快照。你用 status command 把本机状态变成一份可携带的 Markdown 报告。它涵盖就绪度、活跃会话、skill 运行健康、安装健康、待定的治理事件，和链接的工作项目。相关的 commands 让你加入手动的工作项目条目、从 GitHub 同步拉取请求和议题队列，并且在就绪度需要关注时让自动化失败。

ECC 还包含运行和监督代理循环和并行工作的工具。有 commands 去开始和检查受控的代理式循环，配一个 loop-operator agent，它监控卡顿并且介入。有一个持久的 REPL，有模型路由、skill 热加载，和会话分支、搜索、导出、压缩，和度量。有一个 autonomous-loops skill，有给按顺序流水线、拉取请求循环，和基于图的编排的模式，包括防止失控观察者循环的守卫。有一个 worktree 编排脚本，用一个终端多路复用器来做多代理工作。还有一个 harness-audit command，为你的框架设置评分可靠性、评估就绪度和风险，配一个 harness-optimizer agent 去调校它。这些东西强大但有点危险，所以在自主运行任何东西之前，重新看一次安全那一章。

还有一些值得知道的 npm 脚本。有一个完整验证 command，会检查 unicode 安全、验证 agents、commands、rules、skills 和 hooks、运行目录和注册检查，然后运行测试套件。有一个 command 去启动桌面界面。有框架可靠性和风险评分。有一个可观测性就绪度检查。有一个操作者就绪度仪表板。有控制面底座。有一个供应链扫描。有持久的 REPL。还有一个编排状态 command。

那么，重点精华。ECC 2.0 是单个安装之上的操作者层：管理很多会话、看状态和风险，并且编排。Rust 控制平面是真实但 alpha 的，有 dashboard、start、sessions、status、stop、resume，和 daemon commands。操作者命令行界面今天是稳定的，有 consult、install、doctor、repair、list-installed、uninstall，和 Markdown 状态报告。而编排工具存在，但要小心用。
