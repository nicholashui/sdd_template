欢迎收看《Everything Claude Code 完整实战指南》。这是一本书那么长、注重实践的指南，讲解一个叫做 ECC 的项目，ECC 就是 Everything Claude Code 的缩写。你可以把它想成一套为 AI 辅助软件开发而设计的代理框架操作系统。

在整个指南里，你会学到 ECC 是什么、从哪里来、是怎么构建出来的，而最重要的是，怎么在日常工作中真正地运用它。这本书是写给那些从未接触过这个项目、又想从零开始变成一位有信心的操作者的人看的。

在我们继续之前，有一个关于命名的重要提醒。如果你是搜索 ECC，期望找到椭圆曲线密码学，那么这是一个完全不同的项目。简短的 repo 名会重定向到 Everything Claude Code。所以在整本书里，每当我们说 ECC，我们指的都是 Everything Claude Code。这里完全没有密码学成分。它是一套由 agents、skills、hooks 和 rules 组成的系统，用来强化 AI 编程助手。

那这本书是写给哪些人看的呢？是写给那些新用户，他们已经安装了 ECC，或者正在考虑装，但面对 64 个 agents、262 个 skills 和 84 个 commands 时感到无从下手。是写给现有的 Claude Code、Cursor 或者 Codex 用户，他们想要一个清晰的心智模型，而不是一堆配置文件。是写给团队主管，他们正在评估应不应该采用 ECC，以及怎样安全地采用。也是写给任何单纯好奇的人，想了解一套认真的代理框架是怎样设计出来的。

本书不假设你事先懂得 ECC。如果你对终端有基本认识，并且至少用过一款 AI 编程工具，无论是 Claude Code、Cursor、Codex 还是类似的工具，都会有帮助，但你需要的也就这么多。

那么，你应该怎样阅读本书呢？各章节的编排是层层递进的，但每一章也足够独立，可以随意跳读。如果你赶时间，这里有几条捷径。如果你只是想让它跑起来，就看前言，再看安装那一章，然后看日常工作流程那一章。如果你想先理解它，就看背景和架构那两章，然后再深入各个组件。而如果你担心安全问题，就在安装任何会执行 hooks 的东西之前，先看安全那一章。

这里有一幅快速地图，告诉你我们要去哪。第一部分是入门定向，讲解 ECC 是什么以及它为什么会存在。第二部分让它运作起来，包括安装和核心概念。第三部分是本书的核心，也就是六大构建模块：agents、skills、commands、hooks、rules 和记忆，以及 MCP 和上下文管理。第四部分讲怎样操作 ECC，包括日常工作流程、跨框架使用、持续学习，和 token 优化。第五部分讲安全和进阶主题，包括安全、ECC 2.0 和它的命令行界面，以及仪表板和工具。而第六部分是参考资料，有疑难排解、常见问题，和词汇表。

最后一点关于准确性和资料来源。本书里所有内容，都是在直接阅读了 ECC 在 2.0.0 版本的源代码之后写成的。ECC 每个星期都会推出更新，所以 agents、skills 和 commands 的确切数目，连同 command 的名字，都会随时间而变。请务必和实时的 repo 以及里面的指南互相对照。这个项目以 MIT 许可证发布。本书的内容，是为了教学用途，从 ECC 的 repo 和它的文档总结和重新表述出来的。

准备好了吗？我们就从前言和快速上手开始吧。
