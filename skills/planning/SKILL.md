# Skill: Planning

Decompose a goal into a spec and an ordered task list before writing code.

## When to use
- The task is non-trivial or spans multiple files.
- Requirements are ambiguous and need to be made explicit.

## Steps
1. Restate the goal and its acceptance criteria.
2. List unknowns and resolve or flag them.
3. Produce a minimal, ordered task list.
4. Confirm scope before implementing.
